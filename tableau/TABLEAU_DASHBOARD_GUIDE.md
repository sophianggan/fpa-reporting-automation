# Tableau Dashboard Setup — FP&A Reporting Automation

This document describes the three Tableau dashboards built for daily and monthly
reporting by desk heads, FP&A, and the Head of Global Markets.

---

## Data Connection

**Source:** PostgreSQL direct connection (or CSV extracts for demo)

- **Server:** `localhost`
- **Port:** `5432`
- **Database:** `fpa_reporting`
- **Schema:** `gm_fpa`

**Tables / Views to connect:**

| Tableau Data Source Name | PostgreSQL Object           | Grain             |
|--------------------------|-----------------------------|--------------------|
| Monthly Variance         | `vw_monthly_variance`       | Desk × Product × Month |
| Desk Summary             | `vw_desk_summary`           | Desk × Month       |
| Daily P&L Detail         | `vw_daily_pnl_detail`       | Desk × Product × Day |

**Refresh schedule:** Daily at 06:30 via Tableau Server extract refresh
(or manual refresh for Tableau Desktop / Public).

**CSV fallback:** Use the files in `data/processed/` generated by
`python python/export_for_excel.py`

## Dashboard 1: Desk P&L Scorecard (Daily Monitoring)

**Purpose:** Traders and desk heads check this every morning to see
how their MTD P&L compares to the prorated monthly plan.

### Main Visual — Horizontal Bar Chart
- **Data source:** Daily P&L Detail
- **Mark type:** Bar (horizontal)
- **Rows:** `desk_name`
- **Columns:** `SUM(net_pnl_usd)` — MTD cumulative P&L
- **Reference line:** Prorated plan = `plan_pnl × (business_day_of_month / total_business_days)`
  - Calculated field: `[Plan Pnl] * MAX([Business Day Of Month]) / MAX([Total Business Days])`
- **Color:** Conditional — green if actual ≥ prorated plan, amber if within 10%, red if below
  - Calculated field for color:
    ```
    IF SUM([Net Pnl Usd]) >= [Prorated Plan] THEN "On Track"
    ELSEIF SUM([Net Pnl Usd]) >= [Prorated Plan] * 0.9 THEN "Watch"
    ELSE "Behind"
    END
    ```

### Secondary Visual — Heat Map
- **Rows:** `desk_name`
- **Columns:** `product_name`
- **Color:** `SUM(net_pnl_usd) - [Prorated Plan by Product]` (variance)
- **Tooltip:** Shows actual, plan, variance, and trade count
- **Use case:** Instantly spot which product within a desk is dragging performance

### Filters
- `region` (dropdown)
- `asset_class` (multi-select)
- `year_month` (dropdown, default = current month)
- "As of Date" parameter (default = today)

## Dashboard 2: Actual vs Plan vs Prior Trend (Monthly Review)

**Purpose:** Used in the monthly business review by the Head of Markets
and CFO to assess performance trajectory over 7+ years (Apr 2018 – Jun 2025).

### Main Visual — Dual-Axis Line Chart
- **Data source:** Monthly Variance
- **Columns:** `year_month` (discrete)
- **Left axis (lines):**
  - `SUM(actual_pnl)` — solid line
  - `SUM(plan_pnl)` — dashed line
  - `SUM(prior_pnl)` — dotted line
- **Right axis (bar):**
  - `SUM(var_actual_vs_plan)` — cumulative YTD variance
  - Calculated as running sum: `RUNNING_SUM(SUM([Var Actual Vs Plan]))`
- **Color legend:** Actual = blue, Plan = gray (dashed), Prior = light gray (dotted)

### Secondary Visual — Small Multiples
- **Layout:** 2×2 grid, one mini line chart per desk
- **Each chart:** Same 3 lines (actual/plan/prior) over full date range (or trailing 12 months via filter)
- **Title:** Desk name + YTD variance in parentheses
- **Use case:** Compare all four desks' trajectories side by side

### Filters
- `fiscal_year` (default = current year)
- `desk_name` (highlight action — click one desk to focus)
- `asset_class` (dropdown)

## Dashboard 3: RoRWA & Capital Efficiency (Planning)

**Purpose:** Directly supports the Solver-based capital allocation decision.
Presented in the monthly business review alongside the Excel optimizer output.

### Main Visual — Scatter Plot
- **Data source:** Monthly Variance (latest month)
- **X-axis:** `SUM(actual_rwa)` — risk-weighted assets
- **Y-axis:** `SUM(actual_pnl)` — net P&L
- **Mark:** Circle, one per product
- **Size:** `SUM(actual_notional)` — notional traded
- **Color:** `desk_name`
- **Reference line:** 45-degree line representing target RoRWA (12% annualized)
  - Calculated: `[Actual RWA] * 0.12 / 12` (monthly equivalent)
  - Products above this line are capital-efficient
  - Products below are underperforming on capital
- **Labels:** `product_name` (show for all points)
- **Tooltip:** RoRWA, variance vs plan, trade count

### Secondary Visual — Table
- **Top 10 products by RoRWA** (sorted descending)
- **Bottom 10 products by RoRWA** (sorted ascending)
- **Columns:** Product Name | Desk | Actual P&L | RWA | RoRWA | Var vs Plan
- **Conditional formatting:** Highlight rows where Var vs Plan < 0

### Filters
- `year_month` (default = latest month)
- `desk_name` (multi-select)
- `asset_class` (multi-select)

## Calculated Fields Reference

```
// Prorated Monthly Plan (for daily scorecard)
[Prorated Plan] = SUM([Plan Pnl]) * MAX([Business Day Of Month]) / MAX([Total Business Days])

// YTD Cumulative Variance
[YTD Var vs Plan] = RUNNING_SUM(SUM([Var Actual Vs Plan]))

// RoRWA Annualized
[RoRWA Ann] = IF SUM([Actual Rwa]) > 0
              THEN SUM([Actual Pnl]) * 12 / SUM([Actual Rwa])
              ELSE 0 END

// Performance Status (traffic light)
[Status] = IF SUM([Actual Pnl]) >= [Prorated Plan] THEN "On Track"
           ELSEIF SUM([Actual Pnl]) >= [Prorated Plan] * 0.9 THEN "Watch"
           ELSE "Behind" END

// RoRWA Target Line (for scatter plot reference)
[RoRWA Target PnL] = SUM([Actual Rwa]) * 0.12 / 12
```

3. Set permissions: Desk heads = Viewer, FP&A = Explorer, Head of Markets = Viewer

1. Use CSV extracts from `data/processed/`
2. Publish to Tableau Public profile
3. Link from GitHub README
